# Jenkins CI/CD Pipeline — Java Calculator Project

This project contains a simple Java Calculator application built with Maven and tested using JUnit 5. It is designed for use in a Jenkins CI/CD pipeline to demonstrate automated builds, testing, and code coverage reporting.


## CI/CD Pipeline Overview

This project is used in a Jenkins CI/CD pipeline that:
- Pulls code from GitHub
- Runs Maven build and tests
- Publishes JUnit test results

## Features
- Java Calculator with basic operations (add, subtract, multiply, divide)
- JUnit 5 test suite
- Maven build configuration

## 🧠 What I Learned
Through this project, I gained hands‑on experience with how real CI/CD pipelines work in modern DevOps environments. Key takeaways include:
🔹 Jenkins as an Automation Orchestrator
- Learned how Jenkins automates the entire build lifecycle
- Understood how Jenkins triggers jobs, pulls code from GitHub, and supervises each stage
- Saw how Jenkins uses exit codes and test reports to determine build health
🔹 Maven Build Lifecycle
- Practiced using Maven to compile Java code, run tests, and package artifacts
- Understood how mvn clean test package drives the full workflow
- Learned how Maven plugins (like Surefire) integrate into the pipeline
🔹 JUnit Testing + Surefire Reporting
- Built unit tests using JUnit to validate application logic
- Learned how the Surefire plugin discovers, runs, and reports test results
- Saw how Jenkins parses Surefire XML to display pass/fail status and trends
🔹 CI/CD Concepts
- Understood how continuous integration ensures code quality on every commit
- Learned how automated testing prevents broken code from reaching production
- Saw how pipelines enforce consistency, reliability, and repeatability
🔹 GitHub Integration
- Connected Jenkins to GitHub for automated builds
- Learned how version control ties directly into CI/CD workflows
- Practiced structuring a project for clarity and reproducibility
🔹 Documentation & Visualization
- Created a clean, modern architecture diagram to document the pipeline
- Improved my ability to explain technical workflows visually and professionally
- Strengthened my portfolio with a real-world CI/CD example
